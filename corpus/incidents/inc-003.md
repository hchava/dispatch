# INC-003 — orders-fact SLA breach (upstream dependency delayed)

## What happened
orders-raw job was delayed by an executor OOM error on the daily run starting at 01:00 UTC. The job normally completes by 05:30 UTC; it did not complete until 08:17 UTC. orders-fact depends on orders-raw completing by 06:00 UTC (per the contract). The orders-fact job started on schedule but processed an incomplete upstream partition.

## Detection
Freshness alert fired at 08:05 UTC when orders-fact missed its 08:00 UTC SLA window.

## Resolution
1. Identified upstream delay via dependency graph alert (orders-raw missed its 06:00 SLA)
2. Held the orders-fact downstream refresh until orders-raw confirmed complete at 08:17 UTC
3. Triggered orders-fact manual run at 08:22 UTC; completed at 08:51 UTC
4. Notified downstream consumers of the 51-minute delay

## Follow-up
- Added upstream SLA monitoring to orders-fact — alert when orders-raw is >30 min late
- Increased orders-raw executor memory allocation to prevent OOM on large partitions
- Added circuit breaker: if upstream is >2h late, page orders-fact on-call immediately
